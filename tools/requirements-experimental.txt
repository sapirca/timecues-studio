# Optional pip deps for the Phase-1+ experimental MIR sidecars
# (span / beatnet / pitch / panns / cue-extras / lyrics / pattern).
#
# Every server's heavy imports are guarded by try/except, so each
# sidecar starts even without these deps installed — it just reports
# `available=false` and the Initialize-models panel surfaces
# "Deps missing" for that detector.
#
# Usage (after installing core deps from tools/python/requirements.txt):
#   pip install torch torchaudio --index-url https://download.pytorch.org/whl/cpu
#   pip install -r tools/requirements-experimental.txt
#
# Heavy size note: torch (~200 MB CPU / ~2 GB CUDA) is shared with the
# Demucs/All-In-One install — listed at the top of tools/requirements-
# allin1.txt's instructions for the same reason. If you've already
# installed it for stems / All-In-One, you don't need to install it again.

# --- SPAN family -----------------------------------------------------------
# (torch + torchaudio installed separately; see header above)
# Silero-VAD: pulled at first use via torch.hub — no pip dep needed here.
# torchcodec: torchaudio >= 2.9 delegates audio decoding to torchcodec, so
# loading the vocals stem raises "torchaudio … requires torchcodec" without
# it. Needed by the SPAN (silero-vad) detect; harmless on older torchaudio.
torchcodec
# JDCNet: pure-PyTorch port lives in tools/python/jdcnet_torch.py; reads
# the original Keras .hdf5 checkpoint via h5py. Docker bakes the weights
# into the span image; run.sh fetches them into .cache/jdcnet/ on
# bare-metal. We explicitly pin h5py here so the pure-bare-metal path
# doesn't rely on tensorflow / matplotlib pulling it transitively.
h5py>=3.10.0
panns_inference

# --- CUE family ------------------------------------------------------------
BeatNet
# basic-pitch pins tensorflow<2.15.1, which has no Linux wheels for Python
# 3.12+. The env marker keeps modern interpreters from trying to build the
# unsatisfiable TF version (which would take the whole `-r` install down
# with it). pitch_server lazy-imports basic_pitch, so the "Deps missing"
# state is harmless when the package is absent.
basic-pitch ; python_version < "3.12"
# autochord lives in tools/requirements-autochord.txt — it pulls the `vamp`
# C extension which needs the system `vamp-plugin-sdk` lib, so run.sh
# attempts it separately (best-effort, won't block the rest of the family).

# --- LYRICS family ---------------------------------------------------------
openai-whisper
# ctc-forced-aligner (MIT, MahmoudAshraf97/ctc-forced-aligner) — word-level
# CTC forced alignment of a reference transcript. We pin it to
# facebook/wav2vec2-base-960h (Apache-2.0, English-only) so the integration
# stays fully permissively-licensed; the package's DEFAULT MMS_FA model
# is CC-BY-NC and is NOT used by lyrics_server.py.
# Pulls transformers + torchaudio (transformers shared with whisper, torchaudio
# shared with span/beatnet).
#
# MUST install from git: the PyPI name `ctc-forced-aligner` was taken over by
# an unrelated package (deskpai.com) whose API exports `Alignment`/
# `AlignmentSingleton`, NOT the `load_alignment_model` / `generate_emissions`
# / `get_alignments` functions lyrics_server.py is written against. A bare
# `pip install ctc-forced-aligner` pulls the wrong package and fails at import.
ctc-forced-aligner @ git+https://github.com/MahmoudAshraf97/ctc-forced-aligner.git

# --- PATTERN family --------------------------------------------------------
# dtai-locomotif (MIT, KU Leuven) — DTW-warped variable-length motif discovery.
# Pulls numba + matplotlib transitively; numba JIT warm-up is ~15 s on the
# first call after server boot. pattern_server lazy-imports it so absence
# is a graceful "Deps missing" state.
dtai-locomotif
